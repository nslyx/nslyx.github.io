需求操作

仓库迁移

Step.1 首先需要有 原始仓库 & 目标仓库 的操作权限～

Step.2 获取裸库并进入裸库目录，有以下两种情况：

  1. 如果有 Git Server 服务器权限，能够进入到服务器上对应原始仓库的裸库目录中，且该环境下能够执行 git 命令，即满足需求。
  2. 若没有上述权限，至少要有需要迁移的库的读取权限。执行下述命令克隆出完整的裸库并进入。
    git clone --bare git@old-server-url/old-store.git

Step.3 以镜像形式导入新的库地址。在裸库目录中执行下述命令将裸库以镜像写入新仓库地址。
git push --mirror git@new-server-url/new-store.git

Step.4 There is no step four， everything was already done.
