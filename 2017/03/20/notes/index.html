Notes - 开发环境相关

开发相关的杂记。
在本地开发中需要用到很多工具来构建本地的开发环境，包含代码的管理，开发，测试和部署。
管理方面主要是对项目代码的版本控制，对变更记录和保存，目前最常见的版本控制器有 svn(集中式) 和 git(分布式) 。
在电脑上需要有工作区的概念，用来把工作的相关文件代码什么的与平时的其他文件区分开。
不同的开发需要不同的一套工具，选用合适强大的工具对开发效率和开发质量很有意义。下面就从PHP开发切入，随便说说涉及的东西。

GitHub 账号

虽然这个仅仅是众多第三方版本控制服务中的一个，却是自己做开发必须的一个。至少需要有个账号，因为现在越来越多的站点可以或者需要使用它作为第三方登陆。一些开源软件的获取也需要GitHub的第三方认证登陆什么的。像是git的一款开源客户端软件sourcetree就可以登陆github账号免费使用，再如PHP composer 安装 yii2 框架，补充依赖时，也需要有 Github 的 Token。mac 比较好用的包管理软件 Homebrew 也需要 GitHub 的 Token。这些都跟 GitHub 有关而不是其他 GitServer 可以替代的，所以仅仅说要使用 GitServer 可以有多个选择，但是做开发注册个 GitHub 账号还是很有必要的。

Git版本控制器

这里说的是安装在电脑上的客户端软件，因为 GitHub 等 GitServer 比较适合开源项目托管的缘故，所以开发中使用的一些资源很可能是被 Git 托管的，这时候在本地用 Git 获取代码是非常便捷的。并且 Git 也却是是强大而好用的版本控制器，有很多有点在此就不一一赘述了。

SVN版本控制器

SVN 也是一个很常见的版本控制器，或许是因为集中式不同于 Git 的分布式，目前没有遇到过第三方的 SVN Server，基本都是公司在内部自行搭建使用。安装的化，根据需求选择，如果所在团队都在使用 SVN 的话，就很有必要安装一个了。

文本编辑器

编程开发嘛，脚本大多数都支持用文本编辑器直接进行编辑，虽然可能对具体语言的功能不是特别全，但是用来打开查看和略微修改各种脚本都是比较方便的。vim 应该是最强大的文本编辑器了，但对于现在的普通人来说不是特别易用。sublime 算是不错的改版，更加的易用，也支持许多插件，并且有win，linux 和 mac 版本，还是比较通用的。在 win 上还有一个叫做 notepad++ 的文本编辑器也挺好的，不过貌似没有其他系统的版本。

IDE - 集成开发环境

对具体语言的开发支持体验比较好的应该就是 IDE 了，有诸如代码提示，函数搜索，批量替换甚至相关数据库链接，三方库加载等等强大的功能。PHP开发的化推荐 phpstorm 感觉用着挺不错的。

数据库管理软件

目前为止已经有不少类型的数据库，Navicat 客户端还是很不错的，尤其是 premium 版本可以支持多种数据库链接。
