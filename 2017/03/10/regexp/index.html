RegExp - 正则表达式

Mar 10, 2017

Contents

  1. 认知
  2. 概念

详细可学习：正则表达式菜鸟教程。以下皆为摘要，不定期补充更新。

认知

  • 正则表达式是一种文本模式，可用于校验文本
  • 不同语言甚至相同语言的不同版本对正则表达式的支持可能不尽相同，但大同小异
  • 在使用正则的时候最好先测试一下正则的正确性
  • 正则与内置函数可做等效操作时选择函数，大量的正则匹配校验性能牺牲很大
  • 精细正则需要详细规则，多数情况下会选择粗略的匹配
  • 正则写法优化也需要学习，高阶用法更有待深入研究…

概念

  • 普通字符：普通字符包括没有显式指定为元字符的所有可打印和不可打印字符。这包括所有大写和小写字母、所有数字、所有标点符号和一些其他符号。
  • 非打印字符
  • 特殊字符
  • 限定符
  • 定位符
  • 选择？捕获！

  • 反向引用：

    对一个正则表达式模式或部分模式两边添加圆括号将导致相关匹配存储到一个临时缓冲区中，
    所捕获的每个子匹配都按照在正则表达式模式中从左到右出现的顺序存储。
    缓冲区编号从 1 开始，最多可存储 99 个捕获的子表达式。
    每个缓冲区都可以使用 ‘\n’ 访问，其中 n 为一个标识特定缓冲区的一位或两位十进制数。
    可以使用非捕获元字符 ‘?:’、’?=’ 或 ‘?!’ 来重写捕获，忽略对相关匹配的保存。
    反向引用的最简单的、最有用的应用之一，是提供查找文本中两个相同的相邻单词的匹配项的能力。

  • 元字符：包含上述所有字符

  • 运算符优先级
  • 字符簇
  • 确定重复出现
  • 简单表达式：正则表达式的最简单形式是在搜索字符串中匹配其本身的单个普通字符(单字符正则表达式)
  • 字符匹配：句点 (.) 匹配字符串中的各种打印或非打印字符，只有一个字符例外。这个例外就是换行符 (\n)
  • 中括号表达式：] 字符本身需放在首位 和 \ 字符仍然需要转义
  • 替换和分组：替换匹配 | 字符任一侧最大的表达式
  • 非捕获元字符
