inode

发表于 2017-09-04

Jenkins 构建时报错： No space left on device

报错的意思是磁盘上没有空间了，但是上服务器看了看，空间明明还有一堆。。。
这个时候就应该意识到时 inode 不足了，df 查看磁盘空间余量，df -i 查看 inode 余量。

inode 感觉一般默认设置的上限是够用的，如果发生溢满就需要进行追究原因，如果是正常情况不够用了，可以考虑修改上限默认值。很明显，这里并不准备讲正常情况。
在服务器 /var/spool/clientmqueue/目录下有定时任务的输出记录文件，多数情况下 inode 是被这里的文件占用消耗完的。所以在设置定时任务的时候需要注意将不必要的输出销毁。这里的文件一般都可以之间删掉就可以释放很多inode ，没什么影响。不过因为文件数量太多，所以直接 rm -rf * 会因为目录信息太大而执行不了。可以用 lsattr ./|awk -F"/" '{print $2}'|xargs -i rm -rf {} 命令删除，文件多删除执行时间也会比较长。需要注意下 $2 是不是文件名称。
释放完inode 后，上述报错也就自然没有了

查看 inode 数量

for i in /*; do echo $i; find $i | wc -l | sort -nr; done
