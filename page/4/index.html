构建时报错： No space left on device

报错的意思是磁盘上没有空间了，但是上服务器看了看，空间明明还有一堆。。。
这个时候就应该意识到时 inode 不足了，df 查看磁盘空间余量，df -i 查看 inode 余量。

inode 感觉一般默认设置的上限是够用的，如果发生溢满就需要进行追究原因，如果是正常情况不够用了，可以考虑修改上限默认值。很明显，这里并不准备讲正常情况。
在服务器 /var/spool/clientmqueue/目录下有定时任务的输出记录文件，多数情况下 inode 是被这里的文件占用消耗完的。所以在设置定时任务的时候需要注意将不必要的输出销毁。这里的文件一般都可以之间删掉就可以释放很多inode ，没什么影响。不过因为文件数量太多，所以直接 rm -rf * 会因为目录信息太大而执行不了。可以用 lsattr ./|awk -F"/" '{print $3}'|xargs -i rm -rf {} 命令删除，文件多删除执行时间也会比较长。
释放完inode 后，上述报错也就自然没有了

Charles Https 抓包 步骤

发表于 2017-04-13

废话不多说，普通的http抓包也不多说了，简单叙述记录下如何实现 对 https 的抓取，

监听本机请求

需要点击charles的帮助中的 SSLProxying 选项，里面有 Install Charles Root Certificate 选项，点击后就在当前机器上安装了Charles 软件作为代理服务器的 SSL 证书，接下来 需要信任该证书，然后浏览器再访问 https 使用 charles 代理的时候就能信任 charles , 但是并没有结束，需要在 Charles 的 Proxy 菜单中的 SSL Proxying Setting 选项中设置要监听的域名和端口 一般 SSL 端口在 443;

#监听其他机器请求
打开 Charles 的 Help->SSL Proxying ->Install Charles Root Certificate on a Mobile Device or Remote Browser… 选项，然后在需要监听的机器上先配置好普通的 http 监听， 然后 按照提示 用浏览器下载安装 Charles 证书，最后那个Charles 仍然需要配置要监听的域名和端口的。

3、将文件MP_verify_4GgDwWddnaSwTLuY.txt（点击下载）上传至填写域名或路径指向的web服务器（或虚拟主机）的目录（若填写域名，将文件放置在域名根目录下，例如wx.qq.com/MP_verify_4GgDwWddnaSwTLuY.txt；若填写路径，将文件放置在路径目录下，例如wx.qq.com/mp/MP_verify_4GgDwWddnaSwTLuY.txt），并确保可以访问。

公众平台接口域名说明
开发者可以根据自己的服务器部署情况，选择最佳的接入点（延时更低，稳定性更高）。除此之外，可以将其他接入点用作容灾用途，当网络链路发生故障时，可以考虑选择备用接入点来接入。

  1. 通用域名(api.weixin.qq.com)，使用该域名将访问官方指定就近的接入点；
  2. 上海域名(sh.api.weixin.qq.com)，使用该域名将访问上海的接入点；
  3. 深圳域名(sz.api.weixin.qq.com)，使用该域名将访问深圳的接入点；
  4. 香港域名(hk.api.weixin.qq.com)，使用该域名将访问香港的接入点。
