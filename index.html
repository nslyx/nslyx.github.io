计量·规格·换算

无论是电商还是供应链，只要涉及到物料，就一定会涉及到 单位·规格·换算，需要注意的是在任何实操过程中，所有值都是估约值，所以小误差量级后会变成大灾难，所以在物料管理这一情境中，有盘点行为来多次修正偏差，提升精确程度。

计量单位

Fields Thoughts
id 唯一确定一个计量单位便于外联关系
类型为 int(3) 无符号自增 主键，999 个计量单位id 应该足够，所以设为3长度
name 计量单位名称
类型为 varchar(8)，要直观的单位表示: mm,cm,dm,m,km \		 ml,l \ mg,g,kg e.g.
– 以上为核心，以下为拓展 –
alias varchar(32), 逗号连接的多个值，选填项
有些计量单位有别称，可以适当作为 title 属性补充显示
type enum，枚举 volume
计量单位类型，质量单位 \		 长度单位 \ 体积单位 e.g.
pid int(3)，值同 id，无限级分类父子关系刻画方式
此处不表示 parent id，而是表示体系内，同类中，相对小的计量单位的 id
leve int(3)，节点的等级
path varchar(255)，可异步脚本维护
base line 基线，以英文逗号相连的 id，用于描述同类单位由小到大的由来关系
ratio int(9) unsigned，与 pid 指向计量单位的比率，
因 pid 指向的是较小单位，一般都是大于1 的数，但是不见得就是整数，
此处针对简单常见的整数换算率
rctm timestamp
记录创建时间
rutm timestamp
记录更新时间

规格单位属性

规格实例

规格实例依赖计量单位，所以计量单位表是这个表数据的前提，但是为了降低耦合，就不设置外键约束了，但是仍然需要明确关联关系。因为也存在计量单位间那种关系，所以表设计也是在那个基础上拓展改进。

Fields Thoughts
id int(10) unsigned
规格实例的唯一标识，便于外联关系，为了更多的记录所以长度设为10
name varchar(32) 规格实例名称，唯一值，可修改
仅对应一种规格详细
uk char(7) 唯一识别码，确保唯一名称变更后仍然不变
计算核心是唯一，如 id+name+time 的 hash 截取 7 位
unit int(3) unsigned FK
外键约束，关联规格单位的id
– 以上为核心，以下为拓展 –
alias 别名，英文逗号相连的多个值
type 以字串表示类型，-为未设置，0为否，1为是
计量规格 \		 配比单位 \ 销售单位 \ 运输单位
spec varchar(255) 规格详细参数，json 数据
每种单位类型有不一样的数据模型，在被使用后将不能修改，对此字段求 hash 值，以方便规格比对
hash char(32) spec 的 hash 值
rctm timestamp
记录创建时间
rutm timestamp
记录更新时间

需要详细深入的了解下微信支付的完整过程

sd H5 网页支付模式

/**  流程 文字叙述
 * 1. 服务端 生成 图文消息 或 二维码 通过微信客户端 展示给 微信支付用户
 * 2. 微信支付用户 点击图文消息 或 扫描二维码 在微信浏览器中 打开商户 H5 页面
 * 3. 网页内请求生成支付订单
 * 4. 服务端 接收请求 并生成商户订单
 * 5. 在服务端调用  [微信支付]统一下单 Api ，请求微信支付系统 生成预付单，接收 返回的 预付单信息(prepay_id)
 * 6. 生成 JSAPI 页面调用的支付参数并签名，返回支付参数(prepay_id, paySign等)到 微信客户端
 * 7. 用户点击发起支付， JSAPI 接口向微信支付系统 请求支付
 * 8. 微信支付系统 检查支付参数合法性 和 授权域权限，然后返回验证结果 并要求支付授权，即在微信客户端上提示输入密码
 * 9. 用户确认支付，输入密码，提交支付授权到微信支付服务器，由服务器验证授权
 * Alt 并行处理
 * 10. 异步通知商户支付结果
 * 11. 告知微信通知处理结果
 * 12. 微信支付服务器向微信客户端 返回 支付结果，并发微信消息提示，微信客户端展示支付消息给客户
 * 13. 微信跳转回商户 H5 页面，查询商户后台支付结果
 * Alt 支付结果
 * 14. 未收到支付通知，调用查询 API，查询支付结果，微信支付系统 返回支付结果，服务端接收 并 向微信客户端返回支付结果。
 * 15. 商户发货及支付后个性化页面提示
 */

常见命名法

发表于 2017-12-21
  • 下划线命名法(UnderScoreCase)
    • 单词间以下划线_相连
  • 连字符(HyphenNaming)
    • 单词间以连字符-相连
  • 匈牙利命名法
    • 前缀表示类型之类的，分系统和应用两种命名方式
  • 驼峰式命名法(Camel-Case)
    • 小驼峰式命名法（lower camel case）首字母小写
    • 大驼峰式命名法（upper camel case）首字母大写
  • 帕斯卡命名法Pascal
    • 等同大驼峰
